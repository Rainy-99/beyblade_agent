# Balance Report — Beyblade v21

## 1. RPM 經濟

### 有效衰減速率（RPM/s）

`effective_decay()` 計算路徑：`base(22) × 材質係數 × 錘子係數 × 低轉加速曲線`

| 材質 | 正常段衰減 | 含錘子懲罰(×1.15) |
|------|-----------|-----------------|
| 無   | 22.00     | 25.30           |
| 木頭 | 19.80     | 22.77           |
| 鋼鐵 | 23.10     | 26.57           |
| 鈦合金 | 25.96   | 29.86           |

低轉加速曲線（RPM < 25% = 250）：乘數從 1.0（@250 RPM）線性升至 1.4（@0 RPM）。

### 無補充下，800 RPM → 0 所需時間

**計算方法**：
- 800→250 RPM 段：線性，`t = 550 / decay`
- 250→0 RPM 段：指數 ODE，`t = ln(1.4) × 625 / decay ≈ 210.3 / decay`

| 材質 | 800→250 (s) | 250→0 (s) | **總計 (s)** |
|------|------------|----------|------------|
| 無   | 25.0       | 9.6      | **34.6**   |
| 木頭 | 27.8       | 10.6     | **38.4**   |
| 鋼鐵 | 23.8       | 9.1      | **32.9**   |
| 鈦合金 | 21.2    | 8.1      | **29.3**   |
| 鋼鐵+錘子 | 20.7 | 7.9    | **28.6**   |
| 鈦合金+錘子 | 18.4| 7.0   | **25.4**   |

### RPM_PER_SPIN = 85 補充效率

每秒最低需旋轉幾次才能恰好抵消衰減：

| 材質 / 武器 | 衰減 | 單圈補充 | **需要 (轉/s)** | 評估 |
|------------|------|---------|---------------|------|
| 無         | 22.0 | 85      | 0.26          | ✅ 輕鬆維持 |
| 木頭       | 19.8 | 85      | 0.23          | ✅ 最佳耐久 |
| 鋼鐵       | 23.1 | 85      | 0.27          | ✅ 正常操作 |
| 鈦合金     | 26.0 | 85      | 0.31          | ⚠️ 積極操作才行 |
| 鋼鐵+錘子  | 26.6 | **42.5**| **0.63**      | ❌ 幾乎不可能持續 |
| 鈦合金+錘子| 29.9 | **42.5**| **0.70**      | ❌ 無法維持 |

> **結論**：RPM_PER_SPIN = 85 對無武器裝備為合理值。錘子組合（補充 ÷2 + 衰減 ×1.15）在鋼鐵或鈦合金材質下製造了近乎不可能維持轉速的局面，建議調整。

### 極端問題

**問題 1：TankTop 幾乎靜止**
- 最大攻擊下僅 33.8 px/s，玩家打不出「彈飛」的滿足感
- TankTop 使用 `on_knockback()` 時另有 `×0.4` 處理，實際速度更低：33.8 × 0.4 ≈ **13.5 px/s**
- 建議：`knockback_resistance: 3.0 → 2.0`
- 修正後最高：(180 × 2.25) / (4.0 × 2.0) = **50.6 px/s** × 0.4 = **20.2 px/s**（仍有存在感）

**問題 2：Screw + titan + crown@0% 可能飛出邊界**
- 675 px/s，場地有效寬約 1202px，約 1.8s 內橫越
- Screw 有牆壁彈射，邊緣情況可接受，但觀感偏暴力
- 建議：`knockback_resistance: 0.6 → 0.7`，最高降至 578 px/s

**問題 3：RollingStone 難以驅趕**
- 最高 162 px/s 感覺打不動，yet attack=40 對玩家威脅巨大（不對稱）
- 建議：`knockback_resistance: 2.5 → 2.0`，最高升至 202.5 px/s
